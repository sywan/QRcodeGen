Repository Map

src/Recovered application bundle and styling used by Vite.
chrome-extension/Manifest V3 extension popup, background update notice logic, icons, and packaged web app copy.
chrome-web-store/Store listing text and publication graphics.
electron/Desktop wrapper that loads the built web app through a private `qrstudio://` protocol.
android/Capacitor Android scaffold. Generated build folders are intentionally ignored.
scripts/Packaging automation, currently including the Chrome Extension zip builder.
test-assets/Local test fixtures such as sample logos.
dist/Generated web output. Rebuild rather than edit directly.
release/Generated installer/package output. Review before publishing.

Command Notes

`npm run build` is the central command. Chrome, Electron, and Capacitor packaging all depend on the web build being fresh.

Recovery Reality

The local Git repository was repaired after `.git` and the original editable source were lost. The current app source is functional but reconstructed from bundled production output.

If a remote GitHub repository still contains cleaner original source history, inspect it before force-pushing this recovered branch.

Safer Force Push

git push --force-with-lease origin main

Guiding Rule

Preserve history when it contains information that cannot be regenerated. Generated installers can be rebuilt. Hand-written source and release notes deserve extra care.
